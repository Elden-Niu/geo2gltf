========================================
  地理数据转GLTF工具 - 使用说明
========================================

一、文件说明
----------------------------------------
可执行文件位置：dist\地理数据转GLTF工具.exe
文件大小：约112 MB（包含所有必要的依赖库）

二、运行方式
----------------------------------------
1. 直接双击 "地理数据转GLTF工具.exe" 即可运行
2. 首次运行可能需要等待几秒钟解压资源
3. 无需安装Python或任何其他依赖

三、功能说明
----------------------------------------
本工具支持以下功能：

【单文件转换】
- 支持 GeoJSON (.geojson, .json) 格式
- 支持 Shapefile (.shp) 格式
- 可自定义输出目录
- 可配置高度、颜色、透明度等参数

【批量转换】
- 可批量处理整个目录的地理数据文件
- 支持递归搜索子目录
- 每个文件可单独配置颜色（双击文件名）
- 显示转换进度和详细日志

四、使用步骤
----------------------------------------
【单文件转换】
1. 点击"单文件转换"标签页
2. 点击"浏览..."选择输入文件（GeoJSON或Shapefile）
3. 选择输出目录（可选，默认与输入文件同目录）
4. 调整参数：
   - 高度值：地理要素的挤出高度（百分比）
   - 启用自动缩放：自动调整高度以保持合理比例
   - 默认颜色：点击色块选择颜色
   - 透明度：0.0（完全透明）到 1.0（完全不透明）
5. 点击"开始转换"按钮
6. 查看转换日志，等待完成

【批量转换】
1. 点击"批量转换"标签页
2. 选择输入目录（包含多个地理数据文件的文件夹）
3. 选择输出目录
4. 配置扫描选项：
   - 勾选"递归搜索子目录"以包含子文件夹
   - 选择要处理的文件类型（GeoJSON和/或Shapefile）
5. 点击"扫描文件"按钮
6. 在文件列表中双击文件名可为该文件单独配置颜色
7. 调整默认参数（适用于所有文件）
8. 点击"开始批量转换"按钮
9. 查看进度条和日志输出

五、注意事项
----------------------------------------
1. 首次运行可能被Windows防火墙拦截，请允许运行
2. 大文件转换可能需要较长时间，请耐心等待
3. 确保输出目录有足够的磁盘空间
4. Shapefile需要包含完整的相关文件（.shp, .shx, .dbf等）
5. 建议在转换前备份原始数据

六、输出格式
----------------------------------------
- 输出格式：GLTF 2.0 (.gltf)
- 可使用以下软件查看：
  * Blender（免费3D软件）
  * Cesium（在线地理可视化）
  * Three.js（Web 3D库）
  * 支持GLTF格式的各种3D查看器

七、常见问题
----------------------------------------
Q: 程序启动慢怎么办？
A: 首次运行需要解压资源，之后会快很多。建议将exe文件放在固定位置使用。

Q: 转换失败怎么办？
A: 请检查：
   - 输入文件格式是否正确
   - 文件是否损坏
   - 查看日志中的错误提示

Q: 支持哪些坐标系？
A: 支持大多数常见的坐标系，会自动读取文件中的CRS信息。

Q: 可以转换大型数据集吗？
A: 可以，但大型数据集可能需要较长时间。建议使用批量转换功能。
