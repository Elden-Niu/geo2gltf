# GUI 批量转换功能使用指南

## 概述

`geo2gltf_gui.py` 现在支持**单文件转换**和**批量转换**两种模式，通过标签页切换使用。

## 启动GUI

```bash
python geo2gltf_gui.py
```

或者直接双击 `geo2gltf_gui.py` 文件（需要配置Python环境）。

---

## 单文件转换模式

在"**单文件转换**"标签页中，您可以：

1. **选择输入文件**：点击"浏览..."选择单个 GeoJSON 或 Shapefile
2. **选择输出目录**：指定转换后的 glTF 文件保存位置
3. **设置参数**：
   - 高度值：控制模型的挤出高度（百分比）
   - 启用自动缩放：根据地理坐标自动转换为米单位
   - 颜色：选择模型颜色
   - 透明度：设置模型的透明度
4. **开始转换**：点击"开始转换"按钮

### 特点
- 实时日志显示
- 进度条指示
- 转换完成后弹出通知

---

## 批量转换模式

在"**批量转换**"标签页中，您可以批量处理整个目录的地理数据文件。

### 使用步骤

#### 1. 选择目录
- **输入目录**：包含待转换文件的目录
- **输出目录**：转换后文件的保存位置（默认为输入目录后缀 `_output`）

#### 2. 批量转换选项

**递归搜索子目录**
- ☑ 勾选：递归搜索所有子目录中的文件
- ☐ 不勾选：仅处理输入目录中的文件（不包含子目录）

**文件类型**
- ☑ **GeoJSON (.geojson, .json)**：处理 GeoJSON 文件
- ☑ **Shapefile (.shp)**：处理 Shapefile 文件
- 可同时勾选多种类型，也可单独选择

#### 3. 参数设置

与单文件模式相同：
- 高度值（%）
- 启用自动缩放
- 颜色
- 透明度

> **注意**：所有参数在两个标签页之间是**共享的**，在任一标签页修改都会同步到另一个标签页。

#### 4. 开始批量转换

点击"**开始批量转换**"按钮，系统会：
1. 搜索符合条件的文件
2. 显示找到的文件数量
3. 逐个转换文件，并实时更新：
   - 进度标签：当前处理文件名
   - 进度条：总体进度
   - 日志窗口：详细的转换信息

#### 5. 查看结果

转换完成后会显示统计信息：
- 总文件数
- 成功数量
- 失败数量
- 总耗时
- 平均速度

### 批量转换特点

✅ **保持目录结构**：递归模式下，输出目录会保持与输入目录相同的子目录结构  
✅ **实时进度显示**：可以看到当前处理的文件名和进度  
✅ **错误容忍**：某个文件失败不会中断整个批量转换过程  
✅ **详细日志**：每个文件的转换状态、输入输出路径、文件大小等信息  
✅ **统计报告**：批量转换结束后显示完整的统计信息

## 实际应用示例

### 示例 1：批量转换当前目录的所有 GeoJSON 文件

1. 切换到"**批量转换**"标签页
2. 输入目录：选择包含 GeoJSON 文件的目录，例如 `C:\Data\GeoJSON`
3. 输出目录：保持默认或选择其他目录
4. **不勾选**"递归搜索子目录"
5. 勾选"**GeoJSON (.geojson, .json)**"
6. 不勾选"Shapefile (.shp)"
7. 调整参数（高度、颜色等）
8. 点击"**开始批量转换**"

### 示例 2：递归转换所有子目录中的 Shapefile

1. 切换到"**批量转换**"标签页
2. 输入目录：选择顶层目录，例如 `C:\Maps\Regions`
3. 输出目录：指定输出位置，例如 `C:\Output\GLTF_Models`
4. **勾选**"递归搜索子目录"
5. 不勾选"GeoJSON (.geojson, .json)"
6. 勾选"**Shapefile (.shp)**"
7. 调整参数
8. 点击"**开始批量转换**"

输出目录会保持与输入目录相同的子目录结构。

### 示例 3：混合格式批量转换

1. 切换到"**批量转换**"标签页
2. 输入目录：选择包含多种格式的目录
3. **勾选**"递归搜索子目录"
4. **同时勾选** "GeoJSON (.geojson, .json)" 和 "Shapefile (.shp)"
5. 调整参数
6. 点击"**开始批量转换**"

这样会处理所有 GeoJSON 和 Shapefile 文件。

## 常见问题

### 1. 批量转换中某些文件失败了怎么办？

批量转换具有**错误容忍性**，某个文件失败不会影响其他文件的转换。查看日志窗口可以看到具体哪些文件失败以及失败原因。

### 2. 如何停止正在进行的批量转换？

目前批量转换一旦开始无法手动停止，但可以关闭GUI窗口来强制终止。

### 3. 输出文件名是什么？

输出文件名与输入文件名相同，只是扩展名改为 `.gltf`。例如：
- `region.geojson` → `region.gltf`
- `area.shp` → `area.gltf`

### 4. 批量转换时如何保持目录结构？

当勾选"**递归搜索子目录**"时，输出目录会自动保持与输入目录相同的子目录结构。

例如：
```
输入目录结构：
C:\Data\
  ├── Region1\
  │   └── area1.geojson
  └── Region2\
      └── area2.geojson

输出目录结构：
C:\Output\
  ├── Region1\
  │   └── area1.gltf
  └── Region2\
      └── area2.gltf
```

### 5. 参数在两个标签页之间是否共享？

是的，**高度值、颜色、透明度、自动缩放**等参数在"单文件转换"和"批量转换"标签页之间是**共享的**。在任一标签页修改参数，另一个标签页的参数也会同步更新。

## 技术细节

- **支持的输入格式**：GeoJSON (.geojson, .json), Shapefile (.shp)
- **输出格式**：glTF 2.0 (.gltf)
- **坐标系统**：自动处理 EPSG:4326 (WGS84) 坐标系
- **自动缩放**：将经纬度（角度）转换为米单位，基于地球半径计算
- **多线程处理**：转换在后台线程中执行，不会冻结GUI

## 命令行批量转换

如果您更喜欢命令行方式，也可以使用 `batch_convert.py` 脚本：

```bash
# 批量转换当前目录
python batch_convert.py input_dir output_dir

# 递归转换子目录
python batch_convert.py input_dir output_dir --recursive

# 只转换 GeoJSON 文件
python batch_convert.py input_dir output_dir --extensions .geojson .json

# 自定义参数
python batch_convert.py input_dir output_dir --height 20 --color 0,128,255 --alpha 0.7
```

详细信息请参考 [BATCH_CONVERT_README.md](BATCH_CONVERT_README.md)。
